## 一、产品概述

### 1.1 产品定位

一个平台管所有 AI Agent。商户通过本平台:
- 管理多个 Agent 的接入和授权
- 实时接收飞书群信号消息
- 监控 Agent 在线状态（中继通信）
- 编排自动化工作流
- 按套餐计费

### 1.2 核心能力

| 能力 | 说明 |
|:----|:----|
| **多租户** | 每个商户独立管理自己的 Agent、消息、配置 |
| **消息路由** | 飞书群消息 → 按 bot_name 路由到对应商户 |
| **信号系统** | REST API 读写消息，API Key / Master Key 两套认证 |
| **中继通信** | Agent 通过中继收发消息，在线检测 |
| **工作流** | 可视化编排 Agent 任务链（待细化） |
| **多平台** | 后台预留其他 IM 平台接入能力 |
| **计费** | 三档套餐限 Agent 数/消息量 |

### 2.1 三层分工

### 2.3 统一消息格式

各端消息统一转为标准结构后入队列：

```json
{
  "platform": "feishu",         // feishu | telegram | wechat | dingtalk | custom
  "platform_msg_id": "xxx",     // 原平台消息 ID（去重用）
  "group_id": "oc_xxx",
  "group_name": "12FZ程序开发组",
  "sender": {
    "id": "ou_xxx",
    "name": "用户188790",
    "avatar": ""
  },
  "text": "消息内容",
  "msg_type": "text",           // text | image | file | system
  "mentions": ["bot_name"],     // @ 的 bot 列表
  "tags": [],
  "tenant_id": "",
  "raw": {}                     // 原始消息体（适配器填充）
}
```

### 2.4 适配器接口（预留）

Go 核心定义适配器接口，每种通讯平台实现：

```go
type PlatformAdapter interface {
    Name() string                                    // 平台标识：feishu, telegram, custom
    Send(merchantID string, msg OutgoingMsg) error   // 发送消息
    Poll() ([]IncomingMsg, error)                    // 拉取消息（轮询模式）
    RegisterWebhook() (string, error)                // 注册 Webhook（推送模式）
    ParseWebhook(body []byte) (*IncomingMsg, error)  // 解析 Webhook 回调
}
```

**自研聊天系统**按此接口实现适配器后，即可与飞书等平台同等对待，无需改动核心代码。

## 三、数据库设计 (WordPress)

### 3.1 商户表 `wp_fz_merchants`

```sql
CREATE TABLE wp_fz_merchants (
    id            BIGINT AUTO_INCREMENT PRIMARY KEY,
    merchant_name VARCHAR(100) NOT NULL,
    contact_email VARCHAR(200) NOT NULL,
    contact_phone VARCHAR(20),
    plan          VARCHAR(20) DEFAULT 'basic',     -- basic | pro | enterprise
    agent_limit   INT DEFAULT 3,                    -- 套餐限制的 Agent 数
    status        VARCHAR(20) DEFAULT 'active',     -- active | suspended | cancelled
    api_key       VARCHAR(64) NOT NULL UNIQUE,      -- 商户 API Key
    created_at    DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at    DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    expired_at    DATETIME,                         -- 套餐到期时间
    KEY merchant_name (merchant_name),
    KEY plan (plan),
    KEY status (status)
);
```

### 3.2 Agent 表 `wp_fz_agents`

```sql
CREATE TABLE wp_fz_agents (
    id            BIGINT AUTO_INCREMENT PRIMARY KEY,
    merchant_id   BIGINT NOT NULL,
    agent_name    VARCHAR(100) NOT NULL,             -- 如 server-tech
    bot_name      VARCHAR(100) NOT NULL,             -- 飞书 bot 名称
    platform      VARCHAR(50) DEFAULT 'feishu',      -- feishu | telegram | custom
    tags          JSON,                              -- 标签，用于路由
    status        VARCHAR(20) DEFAULT 'active',
    last_seen     DATETIME,                         -- 最近一次通过中继通信时间
    config        JSON,                              -- Agent 配置
    created_at    DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (merchant_id) REFERENCES wp_fz_merchants(id) ON DELETE CASCADE,
    UNIQUE KEY bot_platform (bot_name, platform),
    KEY merchant_id (merchant_id),
    KEY status (status)
);
```

### 3.3 消息表 `wp_fz_messages`

```sql
CREATE TABLE wp_fz_messages (
    id            BIGINT AUTO_INCREMENT PRIMARY KEY,
    agent_id      BIGINT NOT NULL,
    merchant_id   BIGINT NOT NULL,
    message_id    VARCHAR(100) NOT NULL,             -- 飞书消息 ID/去重用
    platform      VARCHAR(50) DEFAULT 'feishu',
    sender        VARCHAR(100) NOT NULL,
    text          TEXT NOT NULL,
    msg_type      VARCHAR(20) DEFAULT 'text',        -- text | image | file | system
    is_read       TINYINT(1) DEFAULT 0,
    tags          JSON,
    created_at    DATETIME NOT NULL,
    received_at   DATETIME DEFAULT CURRENT_TIMESTAMP,
    polled_at     DATETIME,                          -- 最后拉取时间（用于 poll/ack）
    acked         TINYINT(1) DEFAULT 0,              -- 是否已确认
    FOREIGN KEY (agent_id) REFERENCES wp_fz_agents(id) ON DELETE CASCADE,
    FOREIGN KEY (merchant_id) REFERENCES wp_fz_merchants(id) ON DELETE CASCADE,
    UNIQUE KEY message_id (message_id),
    KEY merchant_id (merchant_id),
    KEY agent_id (agent_id),
    KEY is_read (is_read),
    KEY polled_at (polled_at),
    KEY acked (acked)
);
```

### 3.4 工作流表 `wp_fz_workflows`

```sql
CREATE TABLE wp_fz_workflows (
    id            BIGINT AUTO_INCREMENT PRIMARY KEY,
    merchant_id   BIGINT NOT NULL,

    updated_at    DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
```

## 五、工作流系统

### 5.1 触发类型

| 触发类型 | 说明 | 示例 |
|:---------|:----|:-----|
| `message_match` | 消息匹配关键词触发 | 收到"重启" → 执行重启工作流 |
| `cron` | 定时触发 | 每天 3:00 执行清理 |
| `webhook` | 外部 Webhook 触发 | 第三方系统推送通知 |
| `agent_timeout` | Agent 超过中继保活时间触发 | Agent 120 秒未通中继 → 告警 |

### 5.2 工作流步骤

每一步是独立的 Action，可串联：

```json
{
  "type": "notify",           // 通知
  "channel": "feishu",        // 通知渠道
  "target": "商户群",
  "template": "{{agent}} 已离线"
}
```

```json
{
  "type": "call_agent",       // 调用 Agent
  "agent": "server-tech",
  "action": "restart",
  "params": { "service": "nginx" }
}
```

```json
{
  "type": "condition",        // 条件分支
  "if": "{{result.status}} == 'failed'",
  "then": [ { "type": "notify", ... } ],
  "else": []
}
```

### 5.3 工作流引擎

- **Go 核心中间件**负责执行工作流
- 从 WordPress DB 读取工作流配置
- 触发时写入 `wp_fz_workflow_logs`，逐步骤执行
- 支持超时、重试、熔断

## 十、风险与预案

| 风险 | 影响 | 预案 |
|:----|:----|:----|
